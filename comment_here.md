## ✍️ Neuen Kommentar schreiben

**HINWEIS:** Um einen Kommentar hinzuzufügen, bearbeite diese Datei und füge deinen Kommentar in das Format unten ein:

```
### [TAG_NAME]
- **@dein_username** (TT.MM.YYYY): "Dein Kommentar hier..."
```

### Verfügbare Tags:
- `performance` - Geschwindigkeit und Performance
- `design` - Design und Benutzeroberfläche
- `features` - Neue Funktionen oder Feature-Anfragen
- `documentation` - Dokumentation
- `security` - Sicherheitsaspekte
- `speed` - Optimierungen
- `bug` - Gefundene Fehler
- `improvement` - Allgemeine Verbesserungsvorschläge

### Template für deinen Kommentar:

```
### [security]
- **@peter** (05.07.2026): "super perfekt danke oder sicher."
```

**Beispiel:**
```
### [features]
- **@max_dev** (05.07.2026): "Würde gerne Dark Mode sehen!"
```

## 📌 Anleitung:

1. **Username eingeben** - Nutze deinen GitHub-Benutzernamen
2. **Tag auswählen** - Wähle das passendste Tag von oben
3. **Kommentar schreiben** - Schreib dein Feedback
4. **Datum hinzufügen** - Format: TT.MM.YYYY
5. **Commit & Push** - Speichern und hochladen

Dein Kommentar wird dann automatisch auf der README angezeigt! ✨
